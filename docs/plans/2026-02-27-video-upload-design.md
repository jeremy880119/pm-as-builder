# Design: 影音上傳與多平台發布

**Date:** 2026-02-27
**Feature Slug:** video-upload-multi-platform
**Status:** Draft

---

## 問題陳述

Firstory 創作者目前只能上傳音訊。Video Podcast 已成為主流趨勢，創作者需要在 Firstory 內完成影片上傳並同步發布至外部影音平台（優先支援 YouTube，未來擴充 Apple Video、Spotify Video），而不需要切換至其他工具手動操作。

---

## 設計原則

1. **音訊流程零干擾**：沒有影片的創作者，集數建立流程完全不變
2. **漸進展開**：影片與平台設定只在相關條件成立時才顯示
3. **平台設定各自獨立**：每個發布目的地有獨立的 metadata，互不影響
4. **擴充優先**：架構支援未來新增平台，不需改動現有區塊

---

## 媒體模型

| 媒體類型 | 必填 | 發布目的地 |
|---------|------|-----------|
| 音訊 | ✅ | RSS（→ Apple Podcasts、Spotify 音訊等） |
| 影片 | 選填 | YouTube、Apple Video、Spotify Video（未來） |

- 音訊和影片**各自獨立上傳**，無從影片提取音軌的需求（Phase 1 不支援）
- 每個發布平台有**各自的設定與格式要求**

---

## 集數建立流程（4 步驟 Wizard）

### Step 1：上傳媒體

```
┌─────────────────────────────────────┐
│  音訊  [必填]                        │
│  ┌──────────────────────────────┐  │
│  │  拖曳或點擊上傳 MP3 / WAV     │  │
│  └──────────────────────────────┘  │
│                                    │
│  影片  [選填]                        │
│  ┌──────────────────────────────┐  │
│  │  拖曳或點擊上傳 MP4           │  │
│  │  上傳後可同步至影音平台        │  │
│  └──────────────────────────────┘  │
└─────────────────────────────────────┘
```

- 音訊與影片可同時上傳，各自顯示進度條
- 未連接任何影音平台：影片上傳區顯示提示「前往設定連接 YouTube 以發布影片」，不阻擋上傳

### Step 2：基本資訊

現有欄位不變：標題、說明、封面圖、集數編號、季數等。

### Step 3：發布設定

採 **accordion 模式**，每個發布目的地一個區塊：

```
▼ RSS 發布                    （永遠顯示）
  · 發布時間：即時 / 排程
  · 訂閱限制、付費設定等（現有欄位）

▼ YouTube                    （有上傳影片 + 已連接才顯示）
  · 標題（預帶集數標題，可獨立修改）
  · 說明（預帶集數說明，可獨立修改）
  · 縮圖（選填，未上傳則用影片預設縮圖）
  · Playlist（下拉，拉取已連接頻道的播放清單）
  · 能見度：公開 / 不公開列出 / 私人
  · Made for Kids：是 / 否
  · AI 生成內容旗標：是 / 否
  · 發布時間：即時 / 排程（MVP 與 RSS 同步，未來可獨立）

▼ Apple Video                 （未來，有影片 + 已連接才顯示）
▼ Spotify Video               （未來，有影片 + 已連接才顯示）
```

**顯示條件邏輯：**

| 條件 | YouTube 區塊 |
|------|-------------|
| 未上傳影片 | 不顯示 |
| 已上傳影片 + 未連接 YouTube | 不顯示 |
| 已上傳影片 + 已連接 YouTube | 顯示，預設展開 |
| YouTube 授權過期 | 顯示警告 + 引導重新授權，可略過繼續發布 RSS |

### Step 4：預覽 & 發布

顯示本次發布的摘要（各平台設定確認），確認後送出。

## 集數列表：Badge 設計

每集顯示一排平台 badge，**有設定該平台就亮，沒有設定就暗或不顯示**：

```
[ RSS ]  [ YT ]  [ Apple ]  [ Spotify ]
```

| Badge | 亮 | 暗 | 不顯示 |
|-------|----|----|--------|
| RSS | 有設定（含草稿、排程、已發布） | — | — |
| YouTube | 有設定 YouTube 發布 | — | 無影片 / 未連接 |
| Apple | （未來） | — | 未連接 |
| Spotify | （未來） | — | 未連接 |

- 集數歸屬「已發布」或「草稿」tab，由 **RSS 發布狀態**決定（主判斷軸不變）
- YouTube 已發布：badge 可點擊，開啟 YouTube 影片連結

---

## 未來平台擴充原則

1. 新增平台 = 在 Step 3 新增一個 accordion section
2. 各平台設定完全獨立，不影響既有區塊
3. 連接管理統一在「設定 → 整合」頁面
4. 音訊平台（Apple Podcasts、Spotify 音訊）走 RSS，不在此擴充

---

## 明確不做（Phase 1）

- 從影片提取音軌作為 RSS 音訊（Video-first 路徑）
- YouTube 發布與 RSS 發布時間獨立控制
- Apple Video、Spotify Video 連接
- 影片剪輯、字幕、章節標記等製作功能
- 多個 YouTube 頻道連接
