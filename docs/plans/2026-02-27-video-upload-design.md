# Design: 影音上傳與多平台發布

**Date:** 2026-02-27
**Feature Slug:** video-upload-multi-platform
**Status:** Design Approved

---

## 問題陳述

Firstory 創作者目前只能上傳音訊。Video Podcast 已成為主流趨勢，創作者需要在 Firstory 發布音訊後，手動切換至 YouTube Studio 重新上傳影片，流程割裂。

本功能讓創作者在同一集數建立流程中，同時管理音訊與影片，並直接同步發布至外部平台。

---

## 設計決策

### 媒體模型

- **音訊**：必填，發布至 RSS feed
- **影片**：選填，發布至外部影音平台（YouTube 等）
- 音訊與影片各自獨立上傳，無相互依賴
- **不支援**從影片提取音軌（MVP 範圍外）

### 路徑選擇

採用**漸進展開（Progressive Disclosure）**：維持現有分步驟 wizard，影片為可選步驟，平台設定用 accordion 做漸進展開。對沒有影片的創作者零感知。

---

## 集數建立流程（4 步驟 Wizard）

### Step 1：上傳媒體

```
┌─────────────────────────────────────┐
│  音訊  [必填]                         │
│  ┌──────────────────────────────┐   │
│  │  拖曳或點擊上傳 MP3 / WAV      │   │
│  └──────────────────────────────┘   │
│                                     │
│  影片  [選填]                         │
│  ┌──────────────────────────────┐   │
│  │  拖曳或點擊上傳 MP4            │   │
│  │  上傳後可同步至 YouTube        │   │
│  └──────────────────────────────┘   │
└─────────────────────────────────────┘
```

- 音訊和影片上傳互相獨立，可同時進行（各自顯示進度條）
- 未連接 YouTube：影片上傳區顯示小提示「連接 YouTube 以發布影片」，不阻擋上傳
- 未上傳影片：Step 3 的 YouTube 設定完全不顯示

### Step 2：基本資訊

現有欄位不變（標題、說明、封面圖、分類…）。

### Step 3：發布設定

```
▼ RSS 發布                    （永遠展開）
  · 發布時間：即時 / 排程
  · 訂閱限制、付費設定…（現有欄位）

▼ YouTube                    （有上傳影片才顯示）
  · 標題           預帶集數標題，可獨立修改
  · 說明           預帶集數說明，可獨立修改
  · 縮圖           選填；未上傳則用影片預設縮圖
  · Playlist       下拉選單，拉取已連接頻道的清單
  · 能見度         公開 / 不公開列出 / 私人
  · Made for Kids  是 / 否
  · AI 生成內容    是 / 否
  · 發布時間       即時發布 / 排程發布

▼ Apple Video               （未來，有影片 + 已連接才顯示）
▼ Spotify Video             （未來，有影片 + 已連接才顯示）
```

**擴充原則：**
1. 新增平台 = 新增一個 accordion section，不改動其他區塊
2. 各平台設定完全獨立，各自可覆寫標題、說明
3. 連接管理統一在「設定 → 整合」頁面，不在集數建立流程裡
4. 音訊平台（Apple Podcasts、Spotify 音訊）走 RSS，不在此擴充範疇

### Step 4：預覽 & 發布

現有流程，顯示各平台的發布摘要。

---

## 邊界條件（Step 3）

| 情境 | 行為 |
|------|------|
| 未連接 YouTube | YouTube 區塊完全不顯示 |
| 已連接但未上傳影片 | YouTube 區塊不顯示 |
| 已連接 + 已上傳影片 | YouTube 區塊展開顯示 |
| YouTube 授權過期 | 顯示警告 + 引導重新授權；可略過繼續發布 RSS |

---

## 集數列表：Platform Badge

每集顯示一排 badge，**有設定該平台就亮，沒設定就暗或不顯示**（與是否發布無關）。

```
[ RSS ]  [ YouTube ]  [ Apple ]  [ Spotify ]
```

| Badge | 亮 | 暗 / 不顯示 |
|-------|-----|------------|
| RSS | 有設定（含草稿） | — |
| YouTube | 有設定（有影片且填過 YouTube 設定） | 無影片 / 未連接 / 未設定 |
| Apple | （未來） | — |
| Spotify | （未來） | — |

**顏色細節待定義（設計稿階段）。**

### 整合至現有 Tab 結構

集數歸屬哪個 tab，由 **RSS 發布狀態**決定（主判斷軸不變）。
YouTube badge 作為附加資訊顯示在集數卡片上，不影響 tab 分類。

**草稿 tab 範例：**
```
集數標題          日期欄             Badge
有排程草稿        2026/03/05        [ RSS● ]  [ YT● ]
無排程草稿        —                 [ RSS● ]
```

**已發布 tab 範例：**
```
集數標題          日期欄             Badge
已發布集數        2026/03/01        [ RSS● ]  [ YT● ]
已發布集數        2026/02/28        [ RSS● ]
```

## Phase Roadmap

| Phase | 範圍 |
|-------|------|
| **Phase 1（MVP）** | 影片上傳 + YouTube 連接（OAuth）+ YouTube 發布設定 + Badge 顯示 |
| **Phase 2** | Apple Video 支援 |
| **Phase 3** | Spotify Video 支援 |
| **未來** | YouTube 獨立發布時間（不跟 RSS 綁定） |

---

## 明確不做（Phase 1）

- 從影片提取音軌作為音訊
- YouTube 獨立於 RSS 的發布觸發（之後再支援）
- 多個 YouTube 頻道連接
- 影片剪輯、字幕生成等製作功能
