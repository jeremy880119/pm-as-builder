**Feature Slug：** saas-plan-upgrade-downgrade
**版本：** v2.0
**日期：** 2026-02-11
**狀態：** Draft
**前序產物：** 無（起點）

## 概述

### 問題陳述

現有升降級流程 UX 不佳，導致兩個核心問題：

1. **升級轉化率偏低（痛點 8/10）：** 用戶撞到付費功能時無法快速理解價值與升級路徑，跳出率高
2. **降級時缺乏挽留機制（痛點 7/10）：** 降級流程無勸退、無風險揭示，用戶輕易離開

不解決的後果：SaaS Conversion Rate 與 Retention Rate 持續低迷，直接拖累 NSM（每月活躍有感商業化創作者數）。

### 目標用戶

處於「成長撞牆期」的專業創作者（醫師、顧問、知識型講師）。七天收聽 600-6,000，有付費能力但受困於數據盲區、製作繁瑣與變現困難。他們目前的替代方案是手動比較方案頁面或直接聯繫客服。

### 成功指標

| 指標 | 目標 | 對應 NSM Input |
|------|------|---------------|
| 升級轉化率 | 提升 20%+ | Conversion Rate |
| 降級挽留率 | > 30%（看到勸退後放棄降級） | Retention Rate |
| Paywall→升級轉化率 | > 15% | Conversion Rate |

### 策略對齊

| 維度 | 對齊結論 |
|------|---------|
| **ICP** | 撞牆期專業創作者 — 完全吻合。他們是最可能升級付費方案的族群 |
| **NSM** | SaaS 線：MAU x ARPPU x **Conversion Rate** x **Retention Rate**。本功能直接提升後兩項 |
| **Roadmap** | Q4 SaaS「升降級 UX 調整」 |

## 功能範圍

### 核心功能（MVP）

本次重設計 5 個核心元件，建立完整的升降級體驗：

| 元件 | 名稱 | 功能 |
|------|------|------|
| **A** | 帳戶資訊頁 | 顯示當前方案狀態，提供變更方案入口 |
| **B** | 方案選擇頁 | 瀏覽所有方案定價與功能差異，觸發升級或降級 |
| **C** | Paywall Modal | 紅鎖頭觸發，說明功能價值並提供快速升級 |
| **D** | 降級 Modal | D1 軟邊界勸退 + D2 硬邊界攔截 + D3 強制降級暫停上傳 |
| **E** | 升級成功 Modal | 升級完成後展示解鎖功能清單 |

另含：
- **全站 Error Banner：** 系統強制降級觸發硬邊界時，所有頁面頂部顯示不可關閉的提示
- **Feature-Tier Registry：** 功能 x 方案的 Single Source of Truth（所有元件的資料來源）

### 明確不做

- 自助硬邊界處理（硬邊界一律由客服手動處理）
- 智慧方案推薦（不根據使用行為推薦方案）
- Enterprise 聯繫銷售流程（走 Stripe Checkout，同其他方案）
- 管理後台 Registry CRUD（工程盤點後以靜態設定維護）

### 未來考慮

- 自助硬邊界處理（用戶自行選擇保留哪些節目等）
- 智慧方案推薦（根據使用量推薦最適方案）
- Registry 管理後台（PM 可自行新增/修改功能對照）
- 降級後資料保留策略的精細化

## 用戶旅程

**入口：** 帳戶資訊頁（`/account`）「變更方案」，或內容頁紅鎖頭

### Happy Path — 升級

1. 用戶在帳戶頁點擊「變更方案」，進入方案選擇頁
2. 瀏覽各方案定價與功能差異（預設年繳）
3. 選擇目標方案，點擊「升級」（首次用戶顯示「14 天免費試用」）
4. 跳轉 Stripe Checkout 完成付費
5. 返回後顯示升級成功 Modal，列出解鎖功能
6. 點擊「立即體驗」導向首頁

### Happy Path — Paywall 快速升級

1. 用戶在內容頁遇到付費功能，看到紅鎖頭
2. 點擊紅鎖頭，Paywall Modal 說明功能價值與所需方案
3. 點擊「升級到 {plan}」→ Stripe Checkout
4. 升級成功 → Modal E → 首頁

### 分支路徑 — 用戶自行降級

1. 用戶在方案選擇頁點擊較低方案
2. **D1 軟邊界勸退：** 顯示降級後將失去的功能清單（來源：降級影響矩陣）
   - 取消 → 回方案選擇頁（挽留成功）
   - 仍要降級 → 後端執行硬邊界檢核
3. **硬邊界檢核全通過** → Stripe Checkout → 降級排程（當期到期後生效）→ 帳戶頁顯示「待降級」狀態
4. **硬邊界有觸發** → D2 攔截 Modal，列出需客服處理的項目，要求截圖傳客服
   - 「保留目前方案」或「我已截圖」→ 都回方案選擇頁
   - **不擋上傳**
   - 客服處理完成後，用戶再次降級時已處理項目不再觸發

### 分支路徑 — 系統強制降級（付款失敗等）

1. 系統觸發強制降級 → 後端執行硬邊界檢核
2. **硬邊界全通過** → 權限立即變更為目標方案，上傳正常
3. **硬邊界有觸發** → 權限暫不變更，**暫停上傳功能**
   - 所有頁面頂部顯示不可關閉的 Error Banner
   - 點擊 Banner 開啟 D3 Modal（列出需客服處理的項目）
   - 客服處理完成後 → 權限變更 + 恢復上傳 + Banner 消失

### 分支路徑 — 邊界情境

- **Stripe 取消/失敗** → 返回方案選擇頁，方案不變
- **Paywall Modal「查看所有方案」** → 導向方案選擇頁
- **Legacy 用戶** → 只看到升級選項，無降級
- **試用期間取消** → 回到 Free，不產生費用
- **待降級期間又升級** → 取消排程降級，直接升級

## 降級生效規則

| 維度 | 用戶自行降級 | 系統強制降級 |
|------|------------|------------|
| 軟邊界 | 顯示 D1 勸退 | 不顯示（系統直接執行） |
| 硬邊界全通過 | Stripe Checkout → 當期到期後生效 | 權限立即變更 |
| 硬邊界有觸發 | D2 攔截（**不擋上傳**），回方案選擇頁 | 暫停上傳 + 全站 Error Banner → D3 Modal |
| 上傳功能 | 始終正常 | 硬邊界觸發時暫停 |

## 限制與邊界

| 類型 | 說明 |
|------|------|
| Edge Case | Legacy 用戶只能升級，不能降回 Legacy |
| Edge Case | 試用期間取消 → 回到 Free，不產生費用 |
| Edge Case | 系統強制降級 + 硬邊界觸發 → 權限暫不變更，等客服處理 |
| Edge Case | 用戶在待降級期間又升級 → 取消排程降級，直接升級 |
| 業務限制 | 14 天試用每用戶僅一次，所有方案共享 |
| 業務限制 | 用戶自行降級於當期到期後生效 |
| 業務限制 | 硬邊界項目必須客服手動處理，無自助選項 |
