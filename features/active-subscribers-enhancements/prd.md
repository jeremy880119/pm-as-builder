# Feature: 活躍訂閱者名單 - 三項功能新增

**Feature Slug：** active-subscribers-enhancements
**版本：** v1.0
**日期：** 2026-02-23
**狀態：** Draft
**前序產物：** 無（起點）

---

## 概述

### 問題陳述
- 活躍訂閱者名單頁面目前缺乏搜尋與排序功能，且缺少訂閱週期資訊，導致運營人員無法快速定位特定訂閱者或進行資料分析。痛點程度：6/10
- 不解決的後果：運營效率持續低落，需手動逐頁瀏覽查找訂閱者，無法快速回應客訴或進行訂閱趨勢分析

### 目標用戶
- Firstory 內部運營團隊（CS、PM、營運）
- 現有替代方案：手動瀏覽頁面、直接查詢 Stripe Dashboard

### 成功指標
- 定位特定訂閱者的操作時間顯著降低
- 運營人員不再需要切換至 Stripe Dashboard 查看週期資訊

---

## 名詞定義

| 名詞 | 定義 |
|------|------|
| 週期 | Stripe subscription 的 interval 欄位，顯示為「年方案」或「月方案」 |
| Show ID | 節目的唯一識別碼（如 cl2ic0mrn0000q1vdgqpscajx 格式），在 UI 上 hover 才可見 |
| 活躍訂閱者 | 目前有有效 Stripe subscription 的訂閱用戶 |

## 功能範圍

### 核心功能（MVP）

**功能 1：新增「週期」欄位**
- 在「方案」欄位右側新增「週期」欄位
- 資料來源：Stripe subscription 的 interval 欄位
- 顯示值：「年方案」或「月方案」

**功能 2：各欄位排序**
- 所有欄位標題可點擊排序（含新增的「週期」欄位）
- 點擊切換循環：未排序 → 升序 → 降序 → 未排序
- 同時只能對一個欄位排序，預設無排序
- 欄位標題旁顯示排序箭頭圖示（▲▼）

各欄位排序邏輯：

| 欄位 | 升序 | 降序 |
|------|------|------|
| Email | A→Z | Z→A |
| User ID | A→Z | Z→A |
| 節目 | 節目數量少→多 | 節目數量多→少|
| 方案 | LITE→PRO→STUDIO→ENTERPRISE | 反向 |
| 週期 | 月方案→年方案 | 年方案→月方案 |
| 首次訂閱 | 舊→新 | 新→舊 |
| 最近付款 | 舊→新 | 新→舊 |
| 付款次數 | 少→多 | 多→少 |
| 累計金額 | 少→多 | 多→少 |
| Stripe Customer ID | A→Z | Z→A |

**功能 3：搜尋功能**
- 在表格上方新增搜尋欄位，位於「訂閱方案」篩選器右側同一列
- 可搜尋欄位：Email、User ID、節目名稱、Show ID（hover 才可見的 ID）、週期
- 即時過濾（輸入即篩選），case-insensitive 部分比對
- Placeholder：「搜尋 Email、User ID、節目名稱或 Show ID...」

### 明確不做
- 多欄位組合排序
- 進階篩選器（如日期區間、金額區間）
- 資料匯出功能

### 未來考慮
- 多欄位組合排序
- 進階篩選條件
- 搜尋結果匯出為 CSV

## 用戶旅程

**入口：** 活躍訂閱者名單頁面

**Happy Path：**
1. 運營人員進入活躍訂閱者名單頁面
2. 透過搜尋欄輸入關鍵字即時篩選目標訂閱者
3. 或點擊欄位標題進行排序，快速瀏覽資料
4. 查看「週期」欄位確認訂閱者的年/月方案

**分支路徑：**
- 搜尋無結果 → 顯示空狀態提示
- 排序與搜尋同時作用 → 先篩選再對篩選結果排序
- 切換排序欄位 → 前一欄位排序自動取消

## 限制與邊界

| 類型 | 說明 |
|------|------|
| Edge Case | Stripe subscription 無 interval 欄位時，週期欄位顯示為空或「-」 |
| Edge Case | 搜尋關鍵字同時匹配多個欄位時，該筆資料只顯示一次 |
| 業務限制 | 週期資料完全依賴 Stripe API 回傳的 interval 欄位 |
