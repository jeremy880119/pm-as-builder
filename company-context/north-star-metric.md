# Firstory North Star Strategy: The Master Blueprint

**戰略定位：** Shopify for Podcasters (SaaS 工具與雙邊商業市集的混合體)

**目標客群 (ICP)：** 處於「撞牆期」的專業創作者 (醫師、顧問、知識型講師)

**核心哲學：** 拒絕虛榮流量與羞辱性收入，追求「有感」的商業成功。

---

## Part 1: 開場深度討論 (Opening Discussion)

*目的：建立戰略共識，重新定義「影響力」。*

### Q1: 如果我們的工作更有影響力，那會是什麼樣子？

- **願景：** Firstory 的影響力不應只展現在「降低製作門檻」，而應展現在**「提升創作者生存率」**
- **具體定義：** 當我們更有影響力時，專業創作者將不再因為無法變現或過勞而離開市場。他們會因為 Firstory 提供的自動化與變現能力，實現「自我造血 (Self-Liquidating)」，將 Podcast 從「燒錢的興趣」轉變為「獲利的資產」

### Q2: 我們是否清楚產品策略？什麼阻礙了清晰度？

- **策略：** 我們是 **Hybrid Model (混合模式)**。我們既賣鏟子 (SaaS工具費)，也收過路費 (廣告/訂閱分潤)
- **阻礙：** 過去我們過度關注「供給端 (Supply)」的工具體驗，忽略了「需求端 (Demand)」的變現流動性，導致創作者做出了內容卻賣不掉。策略必須修正為**供需同步驅動**

### Q3: 產品驅動 (Product-led) 對我們意味著什麼？

- **定義：** 意味著**「商業媒合的自動化」**
- **標準：** 創作者從「符合變現資格」到「收到第一筆款項」，中間應由系統自動判定與媒合。產品不僅是編輯器，更是自動化的廣告接單機與財務管家

### Q4: 我們現在的投資如何連結到未來的商業表現？

- **邏輯：**
  - 投資 AI (生產力) → 確保庫存穩定 → 提升 **Retention**
  - 投資 AdTech (變現力) → 確保庫存賣出 → 提升 **LTV**
  - 兩者結合，才能建立可持續的訂閱營收與分潤營收

## Part 2: 識別賽局 (Identify the Game)

*目的：確認價值交換邏輯。*

### 賽局判定：雙邊交易遊戲 (Two-Sided Transaction Game)

我們是一個賦能生產力的交易市集。

**價值交換 (Value Exchange)：**

| 端 | 投入 | 交換 |
| --- | --- | --- |
| **創作者端 (Supply)** | 高品質內容 | 「有感收益 (>$15/mo)」或「專業工具 (SaaS)」 |
| **廣告主端 (Demand)** | 預算 | 「精準受眾 (Targeted Audience)」 |

**Firstory 的角色：** 我們是交易所。我們的任務是最大化**交易的流動性 (Liquidity)** 與 **交易的成功率**。

**修正點：** 不再單純看生產力（發幾集），而是看交易結果（賺多少/付多少）。

## Part 3: 好壞指標檢核 (Good vs. Bad NSM)

*目的：設定嚴格濾網，剔除虛榮與無效指標。*

**最終候選：** 每月活躍「有感商業化」創作者數 (Monthly Active Meaningfully Commercialized Creators)

| 檢核標準 | 測試結果 | 關鍵理由 |
| --- | --- | --- |
| **1. 表達客戶價值？** | ✅ Pass | 設定 $15 門檻，確保收入能買一杯咖啡或抵銷月費，而非 $1 的羞辱性收入 |
| **2. 代表願景？** | ✅ Pass | 只有創作者賺到錢/願付費，才代表「事業加速器」願景實現 |
| **3. 是領先指標？** | ✅ Pass | 「回本」是預測續約與留存的最強因子 |
| **4. 可行動？** | ✅ Pass | 逼迫產品優化 AI (降成本)、逼迫業務找廣告 (增收益) |
| **5. 非虛榮？** | ✅ Pass | 嚴格剔除低價方案 ($0.99) 與微薄分潤者 |
| **6. 時間週期？** | ✅ Pass | 使用 **Monthly**，尊重專業節目「雙週更」的高品質節奏，不強迫生產水內容 |

## Part 4: 抽象練習 (Abstract Exercises)

*目的：找出用戶旅程中的「死亡谷」。*

### 信心階梯 (Confidence Ladder) - 修正版

| Level | 階段 | 說明 |
| --- | --- | --- |
| 🪜 **Level 1** | 門檻 | 用戶註冊並開始發布 *(Cost: Low)* |
| 🪜 **Level 2** | 孵化期 (Incubation) | 用戶流量成長，被系統標記為 **"Qualified Creator"** (具備變現潛力的庫存)<br>*風險：這是最危險的階段。如果系統沒能及時將其轉化為變現者，他們就會流失* |
| 🪜 **Level 3** | 承諾期 (Commitment) | 用戶開啟變現功能或升級付費方案 |
| 🪜 **Level 4** | **北極星時刻 (The Break-even Point)** | **用戶單月收益 > 平台訂閱費 (達成損益兩平)**<br>*洞察：這是我們的終極目標。當用戶達到此階梯，Firstory 對他來說就是免費的生財工具* |

## Part 5: 北極星宣言練習 (The Statement Exercise)

*目的：建立四層戰略地圖，絕無省略。*

### 1. 北極星 (North Star)

**宣言：**

> 協助處於「撞牆期」的專業創作者突破瓶頸，將個人影響力轉化為「可自我造血」的可持續商業資產。

**衡量指標 (NSM):**

> **每月活躍「有感商業化」創作者數**
> **(Monthly Active Meaningfully Commercialized Creators)**

*定義：過去 28 天有發布行為，且滿足以下任一條件：*
- **(A) High-Value SaaS:** 付費訂閱 > Tier 2 Plan (e.g., $15/mo)
- **(B) Meaningful Earning:** 獲得廣告/訂閱分潤 > $15/mo

### 2. 輸入指標 (Inputs)

*...這取決於我們是否有能力：*

| Input 1: 供給端 (Supply) | Input 2: 需求端 (Demand) | Input 3: 媒合轉化 (Match) |
| --- | --- | --- |
| **培育具備變現潛力的優質庫存供應** *(Qualified Inventory)* | **建立充足的商業變現流動性** *(Demand Liquidity)* | **高效將潛力股轉化為實質獲益者** *(Conversion Efficiency)* |
| *Metric:* **每月活躍且「達標(Qualified)」的創作者數** *(流量足以接廣的潛力分母)* | *Metric:* **活躍廣告主數 & 平均 eCPM** *(確保錢池夠深，單價夠高)* | *Metric:* **有感轉化率 (Qualified to Meaningful Rate)** *(成功跨越死亡谷的比率)* |

### 3. 機會點 (Opportunities)

*...關鍵槓桿點是：*

| 針對 Input 1 | 針對 Input 2 | 針對 Input 3 |
| --- | --- | --- |
| **降低製作門檻，擴大 Qualified 基數** | **引入外部預算，提高填充率** | **消除創作者對「商業化」的恐懼** |
| *Metric:* **單集製作時間 & 完聽率** | *Metric:* **廣告填充率 (Fill Rate)** | *Metric:* **收益試算器點擊轉化率** |

### 4. 干預措施 (Interventions)

*...具體執行項目：*

| 針對 Input 1 | 針對 Input 2 | 針對 Input 3 |
| --- | --- | --- |
| **AI 自動化流水線 & 搬家優化** | **自助廣告投放平台 (Self-serve Ads)** | **「離回本還差多少」進度條 (Gamification)** |
| *Metric:* **AI 留存率** | *Metric:* **廣告主續單率** | *Metric:* **功能開啟率** |

## Part 6: 架構與輸入公式 (The Structure and Inputs)

*目的：建立數學公式，融合 Hybrid 模式與 Demand 端。*

**NSM = 廣度 (Breadth) × 深度 (Depth) × 頻率 (Frequency) × 效率 (Efficiency)**

由於 Firstory 採用 Hybrid Model，將公式拆分為 **SaaS** 與 **廣告** 兩條業務線：

---

### SaaS 業務線

| 維度 | 指標 | 說明 |
| --- | --- | --- |
| **Breadth1** | MAU (月活躍用戶數) | 平台的活躍用戶基數 |
| **Depth** | ARPPU (每付費用戶平均收入) | 付費用戶的價值深度 |
| **Efficiency** | Conversion Rate (轉化率) | 從免費到付費的轉化效率 |
| **Frequency** | Retention Rate (留存率) | 用戶持續使用與續訂的穩定性 |

**SaaS 公式：** MAU × ARPPU × Conversion Rate × Retention Rate

---

### 廣告業務線

| 維度 | 指標 | 說明 |
| --- | --- | --- |
| **Breadth1** | 廣告進案數 | 進入系統的廣告案件總量 |
| **Breadth2** | 開啟版位數 | 創作者開放廣告投放的版位供給 |
| **Depth** | 廣告單價 | 每次曝光/播放的價值 (eCPM) |
| **Efficiency1** | 人工處理時間 | 從進案到上線的操作效率 |
| **Efficiency2** | 廣告填充率 | 已售出庫存 / 總可用庫存 |
| **Frequency** | 回頭客數量 | 廣告主續投與回購的穩定性 |

**廣告公式：** (進案數 × 開啟版位數) × 廣告單價 × (1/人工處理時間 × 填充率) × 回頭客數量

## Part 7: 練習 - 他山之石 (Build for another product)

*目的：標竿對照驗證。*

### 案例：Shopify (Commerce Platform)

| 項目 | Shopify |
| --- | --- |
| **North Star Metric** | Active Merchants with GMV (活躍且有交易的商家) |
| **Input (Breadth)** | 新註冊並完成開店設定的商家數 |
| **Input (Depth)** | 使用 Shopify Payments 或進階 App 的深度 |
| **Input (Efficiency)** | 商家存活率 (Survival Rate) 與 GMV 成長率 |

**驗證結論：** Firstory 的 Meaningful Commercialized Creators 與 Shopify 的邏輯完全一致——**平台價值 = 生態系中成功存活並獲利的商業體數量**。我們不看「開店數 (Hosting)」，我們看「賺錢商家數 (Commercialized)」。

## Part 8: 收斂與定案 (Converge)

*目的：最終承諾與信心聲明。*

### 決策紀錄 (Decision Log)

- 🚫 **淘汰「每週活躍 (Weekly)」：** 避免懲罰高品質、製作週期長的專業創作者。改用 **Monthly**
- 🚫 **淘汰「>$0 收益」：** 避免 $1 的羞辱性收入導致創作者流失。改用 **>$15 (Meaningful Threshold)**
- 🚫 **淘汰「總下載數」：** 避免虛榮指標掩蓋變現失敗的事實

### 最終信心聲明 (Final Mad Libs)

> 「我會更有信心，確信我們目前的產品策略正在引領 Firstory 邁向可持續的長期增長，只要我觀察到 (A) 用戶行為 的增加...」

> **(A)** [ 處於撞牆期的專業創作者，在平台上實現了「損益兩平 (Break-even)」甚至獲利，將平台視為核心資產 ]
> *(Stuck professional creators achieving break-even or profit on the platform, regarding it as a core asset.)*

> 「...而這可以透過 (B) 具體指標 來衡量。」

> **(B)** [ 每月活躍「有感商業化」創作者數 (Monthly Active Meaningfully Commercialized Creators) ]
> *Definition: Active in last 28 days AND (Paying > Tier 2 Plan OR Earning > $15/mo)*
